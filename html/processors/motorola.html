Motorola Micros
Motorola processors were used in products which I supported while working in the mobile radio industry. Mobile radio projects

The main processor I worked on was the Motorola MC68HC11.

Firstly we used 64k EEPROMs for storing code written in assembler. As the product and its features expanded we moved to 2MB external flash devices using page switching. The HC11 did not have complex peripherals so in order to generate FFSK we needed to drive a GPIO directly via timed interrupt. This was a time when microprocessors and external memory were making way for the microcontrollers we use today.

Orion Emulators were used for debugging. They acted as plug-in ROM replacements. We were unable to step through code, however, as the Orions only had the ability to monitor the data bus and even then only in 64K chunks. Instead, we relied on trace outputs. This became difficult when using paged memory as we needed to select which page the code was in that we were tracing. Logic analysers were used as an (expensive) alternative to the Orions.

Working at this time was a useful transition from university for me as it required an understanding of embbedded architectures (e.g. Intel vs Motorola). Having a background in electronics helped a great deal. Skills such as following schematics, understanding bus interference characteristics, consciousness of memory and processor speed. All things that are taken for granted in systems today.
